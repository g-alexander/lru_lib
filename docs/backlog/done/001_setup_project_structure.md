# Задача 001: Настройка структуры проекта и базовых типов

## 🎯 Цель
Подготовить каркас Rust-проекта, настроить зависимости и определить основные типы данных, которые будут использоваться в библиотеке.

## 📝 Описание
Необходимо:
1. Настроить `Cargo.toml` с необходимыми зависимостями (например, `serde` для персистентности, `parking_lot` или стандартные `Mutex`/`RwLock` для потокобезопасности, `thiserror` для ошибок).
2. Создать базовую структуру модулей в `src/`.
3. Определить основные перечисления (Enums) для режимов работы кэша и типов ошибок.

## ✅ Критерии приемки (Acceptance Criteria)
- [ ] `Cargo.toml` содержит необходимые зависимости.
- [ ] Проект компилируется (`cargo check`).
- [ ] Базовые типы (Error, CacheMode) определены в коде.
- [ ] Нет логики кэша, только структура и типы.

## 🛠 Шаги реализации
1. Редактирование `Cargo.toml`.
2. Создание `src/error.rs` для определения типов ошибок.
3. Создание `src/lib.rs` с объявлением модулей.
4. Определение `CacheMode` (Capacity, TTL, Hybrid).

## 🧪 Требования к юнит-тестам
- Проверка, что базовые типы корректно создаются и компилируются.
- Файл тестов: `tests/setup_test.rs` (или внутри `src/lib.rs`).
