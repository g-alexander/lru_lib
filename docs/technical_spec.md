# Техническая спецификация: Расширение интерфейса кэша и добавление бинарного адаптера

## 1. Обзор изменений
Данная спецификация описывает изменения, необходимые для расширения функциональности библиотеки LRU Cache: добавление методов проверки наличия ключа и размера кэша, а также реализация нового адаптера для сохранения данных в бинарном формате с использованием библиотеки `bincode`.

## 2. Изменения в интерфейсе `Cache<K, V>`

### 2.1. Обновление трейта `Cache`
В трейт `Cache` (файл `src/cache/mod.rs`) добавляются следующие методы:

| Метод | Описание | Возвращаемое значение |
| :--- | :--- | :--- |
| `contains(&self, key: &K)` | Проверяет, существует ли ключ в кэше. | `Result<bool>` |
| `len(&self)` | Возвращает текущее количество элементов в кэше. | `usize` |

### 2.2. Реализация в `LruCache<K, V>`
Методы должны быть реализованы с использованием внутреннего хранилища `MemoryStorage`:

- **`contains`**:
  ```rust
  fn contains(&self, key: &K) -> Result<bool> {
      let storage = self.storage.read();
      Ok(storage.contains(key))
  }
  ```
- **`len`**:
  ```rust
  fn len(&self) -> usize {
      let storage = self.storage.read();
      storage.len()
  }
  ```

## 3. Новый адаптер: `BinaryPersistenceAdapter`

### 3.1. Технологический стек
Для сериализации будет использована библиотека `bincode`, так как она обеспечивает высокую скорость и компактный бинарный формат.

### 3.2. Структура модуля
- **Файл:** `src/persistence/binary_adapter.rs`
- **Трейт:** `PersistenceAdapter<K, V>`

### 3.3. Алгоритм работы
- **`save(path, data)`**:
    1. Открыть файл для записи.
    2. Использовать `bincode::serialize_into` для записи `data` (тип `HashMap<K, CacheEntry<V>>`) в файл.
    3. Обработать возможные ошибки ввода-вывода или сериализации.
- **`load(path)`**:
    1. Открыть файл для чтения.
    2. Использовать `bincode::deserialize_from` для чтения данных.
    3. Вернуть `HashMap<K, CacheEntry<V>>`.
