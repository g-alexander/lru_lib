# Задача 002: Определение основных трейтов и типов данных

## 🎯 Цель
Определить структуру данных для хранения элементов кэша и базовые интерфейсы (traits), которые обеспечат гибкость и расширяемость библиотеки.

## 📝 Описание
Необходимо реализовать:
1. Структуру `CacheEntry<V>`: должна содержать само значение `V`, а также метаданные (время последнего обращения, время создания/истечения TTL).
2. Трейты для управления кэшем (например, `Cache` trait), если это целесообразно для архитектуры.
3. Определение типов ключей и значений с учетом ограничений `K: Hash + Eq + Clone + Serialize + Deserialize<'a>` и `V: Clone + Serialize + Deserialize<'a>`.

## ✅ Критерии приемки (Acceptance Criteria)
- [ ] Определена структура `CacheEntry`.
- [ ] Типы данных поддерживают сериализацию через `serde`.
- [ ] Проект успешно компилируется.
- [ ] Нет реализации логики вытеснения (это будет в следующих задачах).

## 🛠 Шаги реализации
1. Создание/обновление `src/models.rs` или аналогичного модуля для хранения структур данных.
2. Реализация `CacheEntry<V>` с полями `value`, `last_accessed`, `expires_at`.
3. Определение базовых ограничений (Capacity, TTL) как типов или структур.

## 🧪 Требования к юнит-тестам
- Проверка создания `CacheEntry` и корректности его полей.
- Проверка того, что типы реализуют необходимые трейты (например, `serde::Serialize`).
- Файл тестов: `src/models.rs` (модуль тестов).
