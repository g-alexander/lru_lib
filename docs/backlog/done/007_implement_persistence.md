# Задача 007: Реализация персистентности (Persistence)

## 🎯 Цель
Реализовать механизмы сохранения и восстановления состояния кэша на диск, чтобы данные могли быть восстановлены после перезапуска программы.

## 📝 Описание
Необходимо реализовать:
1. **Сохранение (Save):** Сериализация текущего содержимого кэша (ключи, значения и метаданные, такие как время последнего обращения) в файл.
2. **Восстановление (Load):** Чтение данных из файла и заполнение кэша.
3. Использование надежного формата (например, JSON через `serde_json` или бинарного формата через `bincode`) для хранения данных.

## ✅ Критерии приемки (Acceptance Criteria)
- [ ] Состояние кэша успешно сохраняется и восстанавливается.
- [ ] После загрузки кэша метаданные (например, время последнего обращения) сохраняются корректно.
- [ ] Обработка ошибок при чтении/записи файлов (отсутствие файла, неверный формат и т.д.).
- [ ] Проект успешно компилируется.

## 🛠 Шаги реализации
1. Выбор формата сериализации.
2. Реализация метода `save(&self, path: &Path)`.
3. Реализация метода `load(&mut self, path: &Path)`.
4. Тестирование процесса "сохранил -> закрыл -> открыл -> загрузил".

## 🧪 Требования к юнит-тестам
- [ ] Сценарий: сохранение пустого кэша и его восстановление.
- [ ] Сценарий: сохранение кэша с данными и проверка целостности после загрузки.
- [ ] Сценарий: обработка ошибок (несуществующий файл).
- Файл тестов: `tests/persistence_tests.rs`.
