# Задача 005: Реализация комбинированного режима (Hybrid mode)

## 🎯 Цель
Реализовать режим работы кэша, который одновременно применяет ограничения как по количеству элементов (Capacity), так и по времени жизни (TTL).

## 📝 Описание
Необходимо объединить логику из задач 003 и 004. Кэш должен:
1. Принимать параметры `capacity` и `ttl`.
2. Удалять элементы, если:
    * Достигнут лимит `capacity` (используя LRU).
    * Истекло время `ttl` (для конкретного элемента).
3. Обеспечивать корректную работу обоих механизмов одновременно без конфликтов.

## ✅ Критерии приемки (Acceptance Criteria)
- [ ] Кэш соблюдает оба ограничения одновременно.
- [ ] Удаление происходит по любому из условий.
- [ ] Проект успешно компилируется.
- [ ] Реализованы интеграционные тесты, проверяющие оба механизма в связке.

## 🛠 Шаги реализации
1. Модификация структуры кэша для поддержки обоих параметров.
2. Интеграция проверки TTL в логику управления емкостью (при вставке нового элемента стоит также проверить, нет ли уже просроченных элементов, чтобы освободить место).
3. Реализация методов `get` и `put` с учетом обоих условий.

## 🧪 Требования к юнит-тестам
- [ ] Сценарий: достижение лимита емкости при наличии просроченных элементов (должны удаляться в первую очередь).
- [ ] Сценарий: достижение TTL при наличии свободного места в кэше.
- [ ] Сценарий: комбинированное вытеснение (один элемент по TTL, другой по LRU).
- Файл тестов: `src/cache.rs` (модуль тестов).
