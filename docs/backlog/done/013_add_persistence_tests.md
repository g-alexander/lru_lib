# Задача 006: Добавление тестов для бинарного адаптера

## 🎯 Цель
Обеспечить надежность работы нового `BinaryPersistenceAdapter` путем написания интеграционных тестов, проверяющих корректность сохранения и восстановления данных в бинарном формате.

## 📝 Описание
Необходимо добавить в существующий файл тестов или создать новый файл с тестами, которые проверяют полный цикл работы `BinaryPersistenceAdapter`.

**Изменяемые файлы:**
- `tests/persistence_tests.rs`

**Детали реализации:**
1. Создать тестовый сценарий:
    - Инициализировать `LruCache`.
    - Добавить несколько элементов в кэш.
    - Использовать `BinaryPersistenceAdapter` для сохранения кэша на диск.
    - Создать новый экземпляр `LruCache`.
    - Загрузить данные из файла с помощью `BinaryPersistenceAdapter`.
    - Проверить, что данные в новом кэше идентичны данным в старом кэше.
2. Проверить обработку ошибок (например, если файл поврежден или отсутствует).

## ✅ Критерии приемки (Acceptance Criteria)
- [ ] Написаны тесты для `BinaryPersistenceAdapter`.
- [ ] Тесты подтверждают целостность данных после бинарной сериализации/десериализации.
- [ ] Тесты проходят при запуске `cargo test`.

## 🛠 Шаги реализации
1. Открыть `tests/persistence_tests.rs`.
2. Реализовать тестовые функции.
3. Запустить `cargo test`.

## 🧪 Требования к юнит-тестам
- Тесты должны быть изолированными (использовать временные файлы).
- Тесты должны покрывать основные сценарии (позитивный и негативный).

## 🔗 Зависимости
- Требует выполнения: `004_implement_binary_persistence_adapter.md`, `005_register_binary_adapter.md`
- Блокирует: Нет
