# Задача 006: Обеспечение потокобезопасности (Thread Safety)

## 🎯 Цель
Сделать структуру кэша безопасной для использования в многопоточных приложениях, гарантируя корректную синхронизацию данных при конкурентном доступе.

## 📝 Описание
Необходимо:
1. Использовать примитивы синхронизации (например, `std::sync::RwLock` или `parking_lot::RwLock`) для защиты внутренних структур данных кэша.
2. Гарантировать, что операции `get`, `put`, `remove` и `clear` являются атомарными с точки зрения состояния кэша.
3. Обеспечить реализацию трейтов `Send` и `Sync` для основного типа кэша.
4. Предотвратить возникновение deadlock'ов при сложных операциях (например, при удалении элементов во время итерации, если таковая будет реализована).

## ✅ Критерии приемки (Acceptance Criteria)
- [ ] Кэш успешно проходит проверку на `Send + Sync`.
- [ ] При одновременном доступе из множества потоков (чтение/запись) данные остаются целостными.
- [ ] Отсутствуют race conditions в тестах.
- [ ] Проект успешно компилируется.

## 🛠 Шаги реализации
1. Обернуть внутренние структуры данных (HashMap, List) в `RwLock` или аналогичный примитив.
2. Обновить методы `get`, `put`, `remove` для захвата соответствующих блокировок.
3. Проверить корректность захвата блокировок (чтение vs запись).

## 🧪 Требования к юнит-тестам
- [ ] Сценарий: конкурентное чтение из множества потоков.
- [ ] Сценарий: одновременная запись и чтение (проверка отсутствия повреждения данных).
- [ ] Сценарий: интенсивная нагрузка (stress test) с множеством потоков.
- Файл тестов: `tests/concurrency_tests.rs`.
