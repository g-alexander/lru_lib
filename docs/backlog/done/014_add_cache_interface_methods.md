# Задача 002: Добавление методов contains и len в интерфейс Cache

## 🎯 Цель
Расширить интерфейс `Cache<K, V>` новыми методами для проверки наличия ключа и получения количества элементов, что позволит пользователям более эффективно взаимодействовать с кэшем без необходимости извлекать значения.

## 📝 Описание
Необходимо добавить методы `contains` и `len` в трейт `Cache` и реализовать их в структуре `LruCache`.

**Изменяемые файлы:**
- `src/cache/mod.rs`

**Детали реализации:**
1. В трейте `Cache<K, V>` добавить:
    - `fn contains(&self, key: &K) -> Result<bool>;`
    - `fn len(&self) -> usize;`
2. В реализации `impl<K, V> Cache<K, V> for LruCache<K, V>`:
    - `contains`: использовать `self.storage.read().contains(key)`.
    - `len`: использовать `self.storage.read().len()`.

## ✅ Критерии приемки (Acceptance Criteria)
- [ ] Методы `contains` и `len` добавлены в трейт `Cache`.
- [ ] Методы реализованы в `LruCache`.
- [ ] Проект успешно компилируется.
- [ ] Существующие тесты проходят без изменений.

## 🛠 Шаги реализации
1. Добавить определения методов в трейт `Cache`.
2. Добавить реализацию методов в `LruCache`.
3. Запустить `cargo check` для проверки компиляции.

## 🧪 Требования к юнит-тестам
- Создать тест, проверяющий `contains` для существующего и несуществующего ключа.
- Создать тест, проверяющий `len` при добавлении, удалении и очистке элементов.
- Файл тестов: `tests/cache_tests.rs` (или расширить существующие).
