# Задача 004: Реализация режима ограничения по времени жизни (TTL-only mode)

## 🎯 Цель
Реализовать логику кэша, которая управляет временем жизни каждой записи, удаляя их по истечении установленного интервала (TTL).

## 📝 Описание
Необходимо реализовать структуру кэша, которая:
1. Принимает параметр `ttl` (максимальное время жизни записи).
2. При каждом обращении к элементу (`get` или `put`) обновляет время его последней активности.
3. При попытке доступа к элементу проверяет, не истек ли его TTL. Если истек — элемент считается удаленным.
4. В этом режиме не применяется ограничение на общее количество элементов.

## ✅ Критерии приемки (Acceptance Criteria)
- [ ] Элементы удаляются (или становятся недоступными) после истечения TTL.
- [ ] Обращение к элементу успешно сбрасывает его таймер TTL.
- [ ] Проект успешно компилируется.
- [ ] Реализованы тесты на корректность работы времени.

## 🛠 Шаги реализации
1. Добавление логики проверки времени в методы `get` и `put`.
2. Реализация механизма очистки просроченных записей (либо при обращении, либо через фоновый процесс/периодическую проверку).
3. Интеграция хранения времени последнего обращения в `CacheEntry`.

## 🧪 Требования к юнит-тестам
- [ ] Сценарий: элемент доступен до истечения TTL и недоступен после.
- [ ] Сценарий: повторный `get` продлевает жизнь элемента.
- [ ] Сценарий: `put` существующего ключа обновляет его TTL.
- Файл тестов: `src/cache.rs` (модуль тестов).
