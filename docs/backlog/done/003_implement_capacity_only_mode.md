# Задача 003: Реализация режима ограничения по емкости (Capacity-only mode)

## 🎯 Цель
Реализовать базовую логику LRU кэша, которая управляет количеством записей, удаляя наиболее старые при достижении лимита.

## 📝 Описание
Необходимо реализовать структуру кэша, которая:
1. Принимает параметр `capacity` (максимальное количество элементов).
2. Использует эффективную структуру данных для LRU (например, комбинацию `HashMap` и двусвязного списка или `indexmap`).
3. При каждом вызове `get(key)` обновляет позицию элемента в очереди вытеснения.
4. При вызове `put(key, value)` удаляет самый старый элемент, если `capacity` превышен.

## ✅ Критерии приемки (Acceptance Criteria)
- [ ] Кэш строго соблюдает лимит `capacity`.
- [ ] Метод `get` обновляет "свежесть" элемента (LRU работает корректно).
- [ ] Проект успешно компилируется.
- [ ] Реализованы базовые тесты для проверки вытеснения.

## 🛠 Шаги реализации
1. Реализация внутренней структуры данных для управления порядком (LRU order).
2. Реализация метода `put` с проверкой емкости.
3. Реализация метода `get` с обновлением метаданных.
4. Реализация метода `remove`.

## 🧪 Требования к юнит-тестам
- [ ] Сценарий: добавление элементов до достижения лимита.
- [ ] Сценарий: добавление элемента сверх лимита (проверка удаления самого старого).
- [ ] Сценарий: обращение к элементу (get) делает его "новым", предотвращая его удаление.
- Файл тестов: `src/cache.rs` (модуль тестов).
